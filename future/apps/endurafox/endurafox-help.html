EnduraFOX — Help & Reference

EnduraFOX is an in-browser S–N fatigue curve analyzer. It fits multiple fatigue models to experimental stress-life data, selects the best fit by AIC, and reports engineering properties (fatigue limit, scatter index, reliability curves) — with no server, no install, no data leaving your machine.

No data is uploaded. All computation runs locally in your browser using JavaScript. Your test data never leaves your device.

Quick Start

  1. Load data — drag-and-drop a CSV/TXT/TSV file onto the drop zone in the left panel, or click Open in the toolbar. Use Demo to load built-in sample data instantly.

  2. Map columns — the Column Mapping card appears automatically. Confirm which column is Stress (S, MPa) and which is Cycles (N). Click ↺ auto for automatic header detection.

  3. Set material & options — choose the material type and stress ratio R. Adjust the confidence level slider (default 95%).

  4. Select models — tick the fatigue models you want to fit in the Fatigue Models card. The five core models are shown by default. Click Advanced Models ▶ at the bottom of the card to reveal five additional specialist models (nth-Root, Kohout-Věchet, Cardano's).

  5. Run Analysis — click ▶ Run Analysis in the toolbar or at the bottom of the left panel. Results appear on the right, and all plot tabs are populated.

  6. Explore & export — switch between the S–N Curve, Residuals, Reliability, Model Compare, and Data Table tabs. Export a JSON report or save the plot as PNG.

Data Format

EnduraFOX accepts plain-text files: CSV, TSV, or whitespace-delimited TXT. The delimiter is detected automatically.

Minimum requirement

Two numeric columns — one for stress amplitude S (MPa) and one for cycle count N. Headers are optional but recommended for auto-detection.

S_MPa,N_cycles,runout 180,52000,0 165,98000,0 150,210000,0 140,350000,0 135,580000,0 125,1200000,0 115,2600000,0 108,5000000,1 100,10000000,1

Runout (censored) rows

A runout (also called a suspended or censored test) is a specimen that did not fail at the applied stress. Mark them with a third column containing 1 (or any truthy value). If no runout column exists, EnduraFOX auto-detects runouts as points whose N value equals the maximum in the dataset.

Column names EnduraFOX recognises automatically: for stress — s, stress, sa, s_mpa, amplitude; for cycles — n, cycles, nf, life, n_cycles; for runout flag — runout, cens, censored, suspended, r.

Accepted delimiters

Comma, semicolon, tab, or one-or-more spaces are all accepted. Mixed delimiters within a single file are not supported.

Rows that are skipped

  • Blank lines
  • Lines beginning with # (comments)
  • Any row where S ≤ 0 or N ≤ 0 after parsing

Fatigue Models

Tick the models you want to fit. More models = slightly more computation, but all models are fast (< 1 s for typical datasets). See the Fatigue Models section for detailed descriptions.

The five core models (Basquin LS, Basquin MLE, Bilinear, Stromeyer, Log-Normal P-curves) are shown immediately. Click the Advanced Models ▶ toggle to expand five additional specialist models: nth-Root LS, nth-Root MLE, Kohout-Věchet, Cardano's LS, and Cardano's MLE. These are collapsed by default because they require wider data ranges or more data points to converge reliably.

Plot Tabs

S–N Curve

The primary view. Shows all failure data points (circles) and runouts (triangles), the fitted curves for all selected models, the confidence band for the best-fit model, and optionally P10/P50/P90 reliability curves. All Plotly interactive controls are available: zoom, pan, hover tooltips, and legend toggle.

Residuals

Relative residual (S_obs − S_pred) / S_obs × 100 % plotted against predicted S for the best-fit model. A horizontal zero line is shown. Look for random scatter with no systematic trend — patterns suggest the model form is inappropriate for the data.

Reliability

Probability of survival curves at P10, P50, and P90 derived from the best-fit model and the fitted log-normal scatter parameter σlog S. Useful for design-allowable stress determination at a given life.

Model Compare

A ranked table of every fitted model, showing R², RMSE, AIC, and BIC. The row with the lowest AIC is highlighted — that is the model EnduraFOX selects as best fit. Use this tab to understand how much evidence supports each model.

Data Table

The parsed dataset as a scrollable table. Stress values are shown in teal, cycle counts in white. A runout flag column (R) is shown when a runout column was mapped.

Fatigue Models

EnduraFOX implements ten models in two groups. All goodness-of-fit metrics (R², RMSE, AIC) are computed in log₁₀(S) space for comparability — except where noted.

Basquin — Least Squares LS

Equation
S = A · Nb

The classic Basquin (1910) power law, linearised by taking log₁₀ of both sides and solved by ordinary least squares linear regression. Fast, deterministic, and numerically stable.

  • A — stress coefficient (MPa); intercept in log–log space
  • b — fatigue exponent (negative); slope in log–log space
  • σlog S — standard deviation of log₁₀(S) residuals
Limitation: Does not use runout data. Assumes normally distributed errors in log₁₀(S). No endurance limit is modelled.

Basquin — MLE (censored) MLE

Equation (same form, different estimation)
S = A · Nb

The same Basquin power law, but parameters are estimated by Maximum Likelihood Estimation in the log₁₀(N) direction. Runout specimens contribute a survival-probability term to the likelihood, giving a statistically correct treatment of censored data.

  • σN — scatter of log₁₀(N) residuals (fitted)
  • σlog S — propagated scatter in the S direction
Note on AIC: The MLE model's AIC is computed in log₁₀(N) space (because censoring is modelled in N-direction) whereas all other models use log₁₀(S) space. The Model Compare table flags this. Direct AIC comparison with other models should be made cautiously.

Optimised using multi-start Nelder-Mead to avoid local minima.

nth-Root Advanced

Equation
S = a · N−1/n + Sₑ

A three-parameter nonlinear model that adds an asymptotic endurance limit Sₑ to the power law. The exponent is expressed as −1/n (n is the "nth root" parameter) for numerical convenience. Available in two variants:

  • nth-Root LS — minimises SSR in log₁₀(S) space; ignores runouts.
  • nth-Root MLE — maximum likelihood in the log₁₀(N) direction; runout rows contribute a survival-probability term, giving statistically correct censored-data treatment.

Parameters:

  • a — amplitude coefficient (MPa)
  • n — root parameter (fixed or auto-optimised, n > 0.5)
  • Sₑ — endurance limit estimate (MPa)

Uses five-start Nelder-Mead. The n value sub-selector lets you fix n (3, 4, 5, 6, 8, 10) or leave it on Auto to optimise freely. Requires at least 3 failure points. Found under Advanced Models ▶ in the Fatigue Models card.

Bilinear (knee) 2-slope

Equation
S = A₁·Nb₁ (N ≤ Nk) | S = A₂·Nb₂ (N > Nk) [continuous at Nk]

Two power-law segments joined continuously at a knee point Nk. The knee location is found by exhaustive search over all interior data points, choosing the split that minimises total SSR in log₁₀(S).

  • A₁, b₁ — left-segment (high-stress, low-cycle) parameters
  • A₂, b₂ — right-segment (low-stress, high-cycle) parameters
  • Nk — knee cycle count (continuous constraint enforced)

Requires at least 5 failure points (minimum 2 on each side of the knee). Well-suited to steels that exhibit a distinct change in slope near 10⁶–10⁷ cycles.

Stromeyer (endurance limit) Sₑ

Equation
S = A · Nb + Sₑ

The Stromeyer (1914) model adds an additive asymptotic endurance stress Sₑ to the Basquin power law. As N → ∞, the predicted S approaches Sₑ from above. Optimised in log₁₀(S) space with four starting points to avoid the local minimum near Sₑ ≈ Smin.

  • Sₑ — estimated endurance (fatigue) limit (MPa); constrained to [0, Smin)

Requires at least 4 failure points. If the data shows genuine levelling-off behaviour, Stromeyer typically out-competes Basquin by AIC.

Caution: If the data do not include high-cycle runouts, Sₑ may be poorly constrained and its value should be treated with caution.

Log-Normal P-curves P10/90

Not a separate curve-fit model, but a scatter model layered on top of the Basquin LS baseline. The residuals of log₁₀(N) from the Basquin fit are assumed to be normally distributed with standard deviation σlog N.

  • P10 — 90% of specimens survive (lower bound)
  • P50 — median survival (same as Basquin LS)
  • P90 — 10% of specimens survive (upper bound / non-conservative)
Scatter Index
T = N₁₀ / N₉₀ = 10^(2 · z₀.₉ · σlog N)

T is shown in the right panel under Scatter T. Typical values: wrought steel T ≈ 3–6; welded joints T ≈ 10–20; cast iron T ≈ 5–15.

Requires at least 4 failure points. Enable the Reliability curves P10/P50/P90 option to show these on the S–N plot.

Kohout-Věchet 4P wide Advanced

Equation (forward model, fitted in log₁₀N direction)
log₁₀(N) = a + b₁·log₁₀(S) + b₂·log₁₀(S − Sₑ)

A four-parameter model that produces a natural S-shape in log–log space by combining two logarithmic terms — one in S and one in (S − Sₑ). As S approaches the endurance limit Sₑ from above, the second term diverges, pulling predicted life toward infinity. This makes it particularly good at capturing both the LCF regime (steep, high-stress end) and the HCF asymptote in one continuous equation.

  • a — log₁₀(N) intercept
  • b₁ — coefficient of the log₁₀(S) term (main slope)
  • b₂ — coefficient of the log₁₀(S − Sₑ) term (curvature near asymptote)
  • Sₑ — endurance limit (MPa); grid-searched over fractions of Smin

Prediction of S from N is done by bisection on the implicit equation. AIC is computed in log₁₀(S) space (from the inverted residuals). Requires at least 6 failure points and a stress range wide enough to show curvature. Found under Advanced Models ▶.

Cardano's — Least Squares CD Advanced

Equation (forward model, N as function of S)
log₁₀(N) = c − a·(log₁₀S)³ − b·log₁₀S   (a > 0, b > 0)

A cubic polynomial in log₁₀(S) space. The cubic term allows the S–N slope to change sign, naturally producing an S-shaped curve that flattens at both low and high stress. Parameters are estimated by minimising SSR in log₁₀(S) space using five-start Nelder-Mead with an OLS warm-start.

Predicting S from N requires inverting a depressed cubic — EnduraFOX uses the exact Cardano formula:

log₁₀(S) = ∛(−q/2 + √D) + ∛(−q/2 − √D)   where D = (q/2)² + (p/3)³
  • a — cubic coefficient (a > 0 ensures curve bends correctly)
  • b — linear coefficient (b > 0)
  • c — log₁₀(N) at inflection (roughly log₁₀(Ninfl))

The Inflection Nc input in the Fatigue Models card sets the starting guess for c (default 10⁵ cycles). Requires at least 3 failure points. AIC is directly comparable to Basquin LS and other S-direction LS models.

Extrapolation caution: The cubic model can curve sharply outside the data range. Use it for interpolation and comparison within the tested cycle range.

Cardano's — MLE CD MLE Advanced

Likelihood (N-direction, with runout censoring)
log₁₀(S) | N ~ Normal(cardano(a, b, c, log₁₀N), σ²)

The same cubic forward model as Cardano's LS, but parameters are estimated by Maximum Likelihood in the log₁₀(S) direction. Runout specimens contribute a survival-probability term to the likelihood — identical in principle to the treatment used in Basquin MLE and nth-Root MLE.

  • a, b, c — same cubic parameters as the LS variant
  • σN — scatter of log₁₀(S) residuals (fitted simultaneously)
Note on AIC: AIC is computed in log₁₀(S) space, so it is directly comparable to the LS-based models. The Model Compare table does not flag this variant.

Falls back to Cardano's LS if fewer than 4 total points (failures + runouts) are available. Found under Advanced Models ▶.

Model Selection Logic

EnduraFOX selects the best-fit model as the one with the lowest AIC among all successfully fitted models in log₁₀(S) space. All ten models compete on equal terms — MLE-direction models whose AIC is computed in a different space are flagged in the Model Compare table so you can apply judgment.

Rule of thumb (Burnham & Anderson):
ΔAIC < 2 → models are essentially equivalent — prefer the simpler one.
ΔAIC 4–7 → moderate evidence for the lower-AIC model.
ΔAIC > 10 → strong evidence; higher-AIC model has negligible support.

The Model Compare tab shows ΔAIC relative to the best model. Use this to decide whether the added complexity of a 3- or 4-parameter model is justified by your data.

Runouts & Censoring

A runout (also called a suspended test or censored observation) is a specimen that was stopped before failure. It provides the information "the specimen survived at least N cycles at stress S" — a right-censored observation.

Standard least-squares models ignore runouts, which biases the fit toward lower fatigue lives. EnduraFOX handles runouts in two ways:

  1. Auto-detection: if no runout column is mapped, data points with N equal to the dataset maximum are treated as runouts.
  2. Explicit column: map a column containing 1 for runout rows in the Column Mapping card.

When Include runouts in MLE is enabled, all MLE models (Basquin MLE, nth-Root MLE, Cardano's MLE) use the survival probability of censored points in their likelihood function — the statistically correct treatment. The status bar shows the runout count after parsing.

Without runouts: Basquin LS and Basquin MLE produce identical fits because censoring has nothing to contribute. The Insights panel will flag this.

Scatter & Reliability

Fatigue data inherently scatter — two identical specimens tested at the same stress can fail at very different cycle counts. EnduraFOX quantifies scatter using a log-normal model and reports the scatter index T.

Pₓ curve (probability of survival = 1 − x)
S_P(N) = A · Nb · 10z_P · σlog S

Where z_P is the standard normal quantile for probability P and σlog S is estimated from the Basquin LS residuals.

For design, use the P10 curve — it means 90% of specimens survive, providing a conservative allowable stress.

Confidence band vs. P-curves: The confidence band (shaded region) reflects uncertainty in the fitted parameters. The P10/P90 curves reflect specimen-to-specimen material scatter. They are different quantities — the P-curves are typically much wider.

FAQ

Why does the analysis take a moment?

Multi-start Nelder-Mead optimisation (up to 2500 iterations × 5 starts) is run for Stromeyer, nth-Root, Kohout-Věchet, Cardano's, and all MLE variants. For typical datasets (< 200 points) this completes in under one second even with all ten models enabled.

Which model should I use as my design basis?

Use the model with the lowest AIC from the Model Compare tab. If ΔAIC < 2 between two models, prefer the one with fewer parameters (simpler model, less risk of overfitting). For design allowables always use the P10 curve from the Log-Normal scatter model.

My file loads but no data appears — why?

  • Check the Column Mapping card and confirm S and N are assigned to the correct columns.
  • Verify the file contains numeric values (no units text in the data cells).
  • Lines beginning with # are treated as comments and skipped.
  • Rows with S ≤ 0 or N ≤ 0 are dropped silently.

The Bilinear model shows "—" for Fatigue Limit — is that correct?

Yes. The Bilinear model does not explicitly model an endurance limit. The right-segment slope b₂ is typically shallower than b₁, but it continues indefinitely. A rough estimate of the fatigue limit at 10⁸ cycles is shown if the extrapolated value is physically plausible.

Why does the confidence band look narrow?

The confidence band width is determined by σlog S and the selected confidence level (slider). If the data scatter is low (tight cluster around the fitted line), σlog S is small and the band is narrow — which is correct. Increase the confidence level to 99% to widen it.

Can I compare datasets from different tests?

Not directly within one session. Load one dataset, run analysis, export the JSON or save a screenshot, then clear and load the next dataset. A future version may support multi-dataset overlay.

The app runs entirely in my browser — can I use it offline?

Once the page has loaded (which requires an internet connection to fetch Plotly and Google Fonts), the computation is fully local. However, if you reload the page offline those external resources will not be available. For fully offline use, save the page with all assets.
