Help & About

How MeshFOX works, supported file formats, and tips for the porosity-to-simulation workflow.

i

What MeshFOX does

MeshFOX is a browser-based engineering helper for Abaqus foundry simulations. It takes a porosity scan from a meshed model and turns it into a ready-to-include material file with per-region density and Young's modulus — and it cleans the stress include file so cavity elements don't crash the solver. Everything happens locally in your browser; no data is uploaded anywhere.

A

Tool 1 — Material Categorizer

  1. Input file — drop a .inp file with one element per line in the form element_id, value. Lines starting with ** are kept as header comments.
  2. Target — choose what the second column represents: Porosity (%) or Young's Modulus (MPa).
  3. Categories — set a count (2–10) and a binning mode:
    • Uniform — N equal bands across [min, max].
    • Uniform + cutoff — N-1 equal bands from min → cutoff, last band from cutoff → max. Use this when the data is skewed (e.g. mostly low porosity with a long high-porosity tail you want lumped together).
    • Customized — edit every band's lower and upper bound by hand.
  4. Material model — pick Aluminum, Steel, Gray cast iron, Compacted cast iron, Ductile iron, Magnesium, or Custom. Then choose a reduction model:
    • Linear — density and E scale as value × (1−P) per band.
    • Non-linear (power law) — the empirical model E = E₀(1−P)n·k_d. Each material carries its own porosity exponent n (Al 2.3 · Gray Fe 1.5 · CGI 1.8 · SGI 2.1 · Steel 2.5 · Mg 2.7) and the defect-severity factor k_d (Gas 1.0, Mixed 1.3, Shrinkage 1.7) sharpens the falloff for irregular shrinkage voids. Density stays linear in (1−P). Recommended for P < 15%.
    The last band is treated as cavity (effectively zero stiffness). For porosity-target runs, density and E scale as value × (100 − lower) / 100 per band; the last band is treated as cavity (effectively zero stiffness). For E-target runs, each band gets its range-midpoint as its E value.
  5. Export — download Local_Material_Categories.inc (drops straight into your Abaqus job via *include) and optionally Cavity_Porosity.txt (element IDs in the highest-porosity bin).
B

Tool 2 — Stress Cleanup

When cavity elements are removed from the simulation, the initial-stress file still references them and Abaqus errors out. Stress Cleanup splits the stress include into two files — one safe to use, one for inspection.

  1. Cavity element IDs — either auto-load from your most recent categorization (recommended) or upload your own .txt with one integer ID per line.
  2. Stress include file — drop the .inc (typically the *INITIAL CONDITIONS, TYPE=STRESS block). The header row and any *-directive lines are passed through to both outputs.
  3. Result — the panel shows cavity IDs, excluded rows, and included rows, with a split bar for proportion.
  4. Export — Stress_element_Included.inc (use this in your simulation) and Stress_elements_Excluded.inc (kept for the record). Each ends with the count trailer ** The number of elements written: N.
→

Recommended workflow

1
Categorize porosity — load your scan in the Material Categorizer, pick a binning that matches your data, export the .inc. The cavity bin (last category) is captured automatically.
2
Switch to Stress Cleanup — the cavity IDs are already pre-filled in the "From last categorization" panel. Drop in your stress .inc, hit Process, download the included file.
3
Add both files to your Abaqus job — include the material file, swap the cleaned stress file in. Run.

FAQ

Does my file leave the browser?

No. MeshFOX runs entirely in your browser — there's no server. Refreshing the page wipes everything.

What's the difference between Uniform and Uniform + cutoff?

Uniform divides the full range into equal slices. Uniform + cutoff puts most of the resolution where your data lives (e.g. 0–20% porosity), and lumps everything past the cutoff into one final bin (e.g. 20–100% as "cavity").

How big a file can it handle?

Tested with ~30k elements with no slowdown. Multi-million-element scans should still work but may take a few seconds to parse.

Why is the last category special?

For porosity-target runs, the last bin is treated as cavity: density falls to a tiny placeholder and E to 1.0. Use Stress Cleanup to strip these element IDs from any initial-stress file before running the job.
